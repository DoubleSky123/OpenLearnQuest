Background & Motivation

Algorithm education presents a persistent challenge in computer science: concepts like linked lists, trees, and graphs are inherently interconnected, yet most educational tools treat them in isolation. A comprehensive literature review of leading platforms revealed a critical market gap:

  • General learning platforms offer engagement mechanics but lack algorithm depth
  • Algorithm practice platforms offer depth but are not designed for novice learners
  • Educational games teach programming syntax but not data structure reasoning

No existing platform combines gamified interaction with adaptive prerequisite routing for algorithm concept learning.

Problem Statement

When students learn a complex algorithm (e.g., doubly linked list operations), they frequently encounter prerequisite gaps mid-learning — discovering they do not understand singly linked lists or pointer basics. Current platforms provide no structured mechanism to detect this gap and route students to remedial content without disrupting their learning flow.

Research Question: How can research-driven iterative design produce an adaptive, gamified learning module that scaffolds algorithm education and enables cross-topic prerequisite routing?

1
Initial Prototype — Basic Drag-and-Drop
A single-page app where students drag pseudocode blocks into an assembly area to perform singly linked list operations. Dark theme, no feedback, no distractors, no scaffolding. Confirmed the core mechanic was learnable but too abrupt for novice users.
Driven by: BlockList paper + initial feasibility
2
Three-Mode Architecture — Tutorial → Training → Challenge
Literature review showed that one-size-fits-all interfaces fail novice learners. Introduced progressive scaffolding: Tutorial (step-by-step hints, fixed exercises), Training (on-demand hints, random values), and Challenge Mode (three difficulty levels, procedurally generated questions with distractor blocks and error taxonomy feedback).
Driven by: Brilliant.org scaffolding model + CogBooks adaptive design
3
Tutorial Intro Redesign — Concept-First Onboarding
Instructor feedback revealed students entering Tutorial had no conceptual baseline. Replaced the welcome popup with a 4-slide concept introduction (linked list structure, node anatomy, HEAD & NULL, key terms) plus a mandatory 4-question quiz. The drag-and-drop mechanic was replaced with fill-in-the-blank to reduce cognitive load in the learning phase.
Driven by: Instructor feedback + Brilliant concept-first design
4
Gamification Layer — Pet Companion, XP, Stars, Lives
Analysis of Duolingo and Prodigy revealed that long-term engagement requires emotional investment beyond task completion. Added a pixel husky companion (Algo) with 5 evolution stages driven by a shared XP pool, a star rating system (0–3 stars based on errors), a lives counter, and a game timer. The path-style mode navigation replaced flat card selection.
Driven by: Duolingo + Tamagotchi engagement research
5
Doubly Linked List Module + Adaptive Cross-Module Routing
Added DLL as the second module with bidirectional node visualization (⇄ arrows, prev/next pointer states) and a new error type: Broken Prev. Implemented cross-module adaptive logic: when ≥ 2 Type-A errors (pointer errors shared with SLL) occur in a DLL session, a non-disruptive suggestion modal appears, framing the return to SLL as a player-initiated choice rather than a system redirect.
Driven by: CogBooks adaptive routing + "player choice" design principle
6
UI Unification, Sort Mode + Module Path Expansion
Instructor usability testing identified font size and dark theme as friction points. Converted all screens to a unified light theme, doubled all font sizes, and replaced drag-and-drop with click-to-place in Tutorial. Added a standalone Sort Linked List mode with live pseudocode highlighting. Module selection now displays a winding path with locked future modules (Sorting, Tree, Graph), communicating the platform's broader vision.
Driven by: Instructor feedback sessions + Duolingo path navigation

Error Taxonomy (Educational Design)

Six error categories map to distinct cognitive misconceptions, enabling targeted feedback:

Lost Reference Off-by-One NULL Pointer Self-Loop Memory Leak Broken Prev (DLL)

Errors are classified as Type A (shared with SLL — triggers adaptive routing) or Type B (DLL-specific — targeted hint only). Distractors are constrained to single lines, consistent variable names, and semantically meaningful wrong answers.

Conclusions

This project demonstrates that a research-driven iterative process — grounded in literature review and instructor feedback — can produce an educationally principled, engaging gamified learning module. Key contributions include:

  • A six-category error taxonomy that maps game errors to real CS misconceptions
  • A cross-module adaptive routing design that preserves learner autonomy
  • A three-tier scaffolding architecture that bridges the novice-to-practitioner gap
  • A modular platform foundation extensible to Sorting, Tree, and Graph algorithm games
Platform vision: A knowledge dependency graph of algorithm games where students can always find — and choose — the prerequisite they need, in the form of a game they want to play.

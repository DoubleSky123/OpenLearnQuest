Figure 1 — Design evolution
Gap 1
No conceptual baseline — novices dropped into exercises with no prior knowledge
Solution
Full scaffolding layer introduced:
Intro (concept slides + mandatory quiz) → Tutorial (guided) → Training (hints on demand) → Challenge (independent)
Brilliant [1], CogBooks, instructor feedback
Gap 2
Basic operations alone did not reflect real CS coursework or interview expectations
Solution
Challenge Level 3 expanded to classic algorithm problems: Reverse Linked List, Detect Cycle, Merge Two Lists, Sort List
LeetCode problem set analysis [2]
Gap 3
No mechanism to sustain motivation beyond individual task completion
Solution
Pixel pet companion (Algo) with 5 evolution stages, experience points, star ratings per question, lives, timer, and a winding progress path
Duolingo [1], StudyGotchi [4]
Gap 4
Single module with no way to detect or address prerequisite knowledge gaps
Solution
DLL module added with bidirectional visualization. Repeated pointer errors trigger an adaptive suggestion to return to SLL — framed as a player choice, not a forced redirect
BlockList [3], design principle
Gap 5
Dark theme and small fonts created unnecessary friction for learners
Solution
Unified light theme across all screens, doubled font sizes, click-to-place replacing drag-and-drop in Tutorial, winding module path showing locked future modules
